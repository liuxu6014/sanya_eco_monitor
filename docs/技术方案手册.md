## 1. 项目概述与资产盘点

### 1.1 项目背景

三亚市天涯区农业农村局智慧农业生态监测平台，面向天涯区农业生产环境，集成多类型物联网传感设备，实时采集气象、墒情、虫情、孢子、水质、雨量、径流等生态数据，通过可视化大屏实现统一展示、分析与预警，并通过 AI 大模型生成专业农业生态分析报告，定期产出季度/年度评估报告。

### 1.2 核心目标

| 目标 | 说明 |
|------|------|
| 实时监测 | 每30分钟自动采集全部设备数据，存储至本地数据库 |
| 可视化展示 | 深色科技感大屏，GIS 地图 + 多模块看板，全指标实时展示 |
| 数据分析 | 趋势分析、周/月聚合统计、面源污染负荷核算、虫情密度指数 |
| AI 智能报告 | 调用 DeepSeek 大模型对监测数据进行专业分析，自动生成文章报告 |
| 报告导出 | HTML / Excel / Word 多格式，按季度/年度自动生成评估报告 |
| 设备管理 | 远程控制设备参数、阈值告警配置、设备在线状态监控 |

### 1.3 硬件设备清单

| 序号 | 设备名称 | 数量 | 核心监测指标 | 接入平台 | 状态 |
|-----|---------|------|------------|---------|------|
| 1 | 智能虫情测报灯 | 1台 | 捕获数量、虫种分类（甜菜白带野螟、大螟等）、设备图片 | zhnlkj.com（编号: 202603172301）| 已接入 |
| 2 | 孢子捕捉仪 | 1台 | 孢子数量、显微图像、自动更换载玻带 | zhnlkj.com（编号: 202603172302）| 已接入 |
| 3 | 农田排水水质及面源污染负荷监测系统 | 1套 | 高锰酸盐指数（COD）、氨氮（NH₃-N）、总磷（TP）、总氮（TN） | iot.whxph.com（设备ID: 16116030）| 协议对接中 |
| 4 | 4G 雨量计 | 3台 | 小时雨量、日累计雨量 | 协议待提供 | 待接入 |
| 5 | 地表径流监测系统 | 6套（含雷达流速仪 + 水位计） | 流量（m³/h）、水位（cm）、流速（m/s） | 协议待提供 | 待接入 |

**虫情灯 / 孢子仪高级功能（zhnlkj.com 平台接口支持）**：

- **远程控制**：下发工作时间（workTime）、采样间隔（sample）、风机转速（fanStrength）等参数
- **阈值告警**：按虫种名称设定 limitMax/limitMin，触发时向指定渠道推送告警

### 1.4 地理空间数据（GIS）

项目提供 **`.gdb` 格式地理数据库文件**，包含：
- **水源林监测点**：地理坐标、边界多边形
- **橡胶林监测点**：地理坐标、边界多边形
- **6个 20m×20m 人工样方**：生物多样性调查样地坐标

用途：在大屏地图上精准标定设备位置、绘制监测区边界、叠加 GIS 分析结果。

**解析方案**：使用 `geopandas` + `fiona` 读取 `.gdb` 文件，提取 GeoJSON 格式坐标，由前端 react-leaflet 渲染多边形边界。

## 3. 数据库设计

### 3.1 设计原则

- 使用 SQLite 轻量级数据库，生产环境可切换 PostgreSQL（仅修改 `DATABASE_URL`）
- 所有表保存 `raw_data` (JSON) 字段，存储原始 API 响应，便于数据溯源
- 使用 SQLAlchemy 2.0 异步 ORM，支持高并发查询

### 4.2 双平台 Token 缓存机制

系统对接两个独立 IoT 平台，各自维护独立的 Token 缓存：

```python
# collectors/base.py
# 平台A：zhnlkj.com — 虫情/孢子/气象/墒情/苗情
_zhnl_cache = {"token": None, "expires_at": 0}

# 平台B：iot.whxph.com — 水质/面源污染
_whxph_cache = {"token": None, "expires_at": 0}

async def get_zhnl_token() -> str:   # zhnlkj.com JWT，30min
async def get_whxph_token() -> str:  # iot.whxph.com JWT
```

**zhnlkj.com 认证**：
- POST `https://zhnlkj.com/iotSmasrt/login` → `data.token`
- 请求头：`Authorization: {token}`（无 Bearer 前缀）
- 证书：自签名，`verify=False`

**iot.whxph.com 认证**（待确认登录路径）：

- 候选路径：`/agriculture/login` / `/environment/login` / `/hydrology/login`
- 前端 JS 揭示：baseURL 动态拼接为 `http://{IP}:{PORT}`
- 已发现的数据接口：`/datas/`、`/device/`、`/intfa/queryData/`、`/screen/datas`

### 7.3 面源污染负荷核算

**核心公式**：

$$污染负荷(kg) = 污染物浓度(mg/L) \times 径流量(m³) \times 0.001$$

**计算流程**（`services/pollution_calc.py`）：

```python
class PollutionCalculator:
    """面源污染负荷计算服务"""

    @staticmethod
    def calculate_load(concentration_mg_l: float, volume_m3: float) -> float:
        """
        污染负荷(kg) = 浓度(mg/L) × 径流量(m³) × 0.001
        0.001 = mg/L × m³ → kg 的单位换算系数
        """
        return round(concentration_mg_l * volume_m3 * 0.001, 4)

    async def compute_daily_load(
        self, db: AsyncSession, target_date: date
    ) -> dict:
        """
        将当日水质浓度与当日地表径流量在时间轴上对齐积分，
        计算 TN、TP 的日污染负荷。
        """
        # 1. 获取当日各时间段水质浓度（TN/TP, mg/L）
        water_records = await self._get_water_records(db, target_date)

        # 2. 获取当日各时间段径流量（m³）
        runoff_records = await self._get_runoff_records(db, target_date)

        # 3. 时间轴对齐 + 梯形积分
        tn_load = 0.0
        tp_load = 0.0
        for wq, rf in self._align_records(water_records, runoff_records):
            tn_load += self.calculate_load(wq.total_nitrogen, rf.cumulative_volume)
            tp_load += self.calculate_load(wq.total_phosphorus, rf.cumulative_volume)

        return {
            "date": str(target_date),
            "tn_load_kg": tn_load,
            "tp_load_kg": tp_load,
        }
```

**报告指标**：

| 统计维度 | 说明 |
|---------|------|
| 日/月/年 TN 污染负荷 (kg) | 总氮流失量 |
| 日/月/年 TP 污染负荷 (kg) | 总磷流失量 |
| 环比变化 | 本期 vs 上期，绝对值 + 百分比 |
| 同比变化 | 本期 vs 去年同期 |

### 7.4 虫情密度指数

```python
def calculate_insect_density(
    total_count: int,
    trap_area_m2: float = 1.0,
    period_days: int = 1,
) -> float:
    """虫口密度 = 捕获总数 / 面积 / 天数 (只/m²/天)"""
    return round(total_count / trap_area_m2 / period_days, 2)
```

### 11.1 开发环境启动

```bash
# 后端
cd backend
uv sync
uv run uvicorn main:app --host 0.0.0.0 --port 8001 --reload

# 前端
cd frontend
npm install
npm run dev   # → http://localhost:5175
```

### 11.2 环境变量配置（backend/.env）

```bash
# 平台 A：zhnlkj.com
```

### 11.3 生产环境建议

| 项目 | 建议 |
|------|------|
| 数据库 | 切换 PostgreSQL，配置每日备份 |
| Web 服务 | Nginx 反向代理，前端 `npm run build` |
| 进程管理 | systemd 管理 uvicorn，配置自动重启 |
| GIS 文件 | `.gdb` 存放于 `backend/data/gis/`，启动时解析缓存 |
| 定时报告 | APScheduler 每季度末自动触发 `generate_quarterly_report()` |
| 日志 | `logging.handlers.RotatingFileHandler`，按天滚动 |

### 11.4 Nginx 参考配置

```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        root /var/www/sanya-eco/dist;
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:8001;
        proxy_set_header Host $host;
        proxy_read_timeout 120s;    # AI 分析接口需要较长超时
    }
}
```

## 附录 C：面源污染负荷计算公式推导

$$L_{TN} = \sum_{i=1}^{n} C_{TN,i} \times V_i \times 10^{-3}$$

$$L_{TP} = \sum_{i=1}^{n} C_{TP,i} \times V_i \times 10^{-3}$$

其中：
- $L$ = 污染负荷（kg）
- $C$ = 污染物浓度（mg/L）
- $V$ = 对应时段径流量（m³）
- $10^{-3}$ = 单位换算系数（mg·m³/L → kg）

**注意**：水质监测时间点与径流监测时间点需在时间轴上对齐插值（线性插值），再对各时段梯形积分，得出日/月累计污染负荷。
