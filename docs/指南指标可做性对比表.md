## 七、建议对甲方的表述方式

为避免后续验收争议，建议对外统一表述为：

1. 当前平台已具备水土流失、农业面源污染、虫害发生等指标的在线监测与阶段评估能力。
2. 水源涵养、污染削减能力等指标仅作为“现有基础可继续补强项”展示，不在本期单独承诺为正式验收指标。
3. 碳汇、入侵植物、重要栖息地、珊瑚礁等指标因缺少专项监测数据，本期前端和报告中不纳入承诺展示。
4. 对于可做但仍属监测型估算的指标，统一采用“监测型估算结果”或“阶段评估结果”表述。

## 八、管理闭环与报告表达建议

除结果指标外，《指南》还关注监测数据是否真正服务于管护决策、预警处置和成效复核。基于当前平台基础，建议在自动化报告中同步加入“监测发现、风险判断、处置措施、结果反馈”四段式表达，使数据结果和管理动作形成闭环。

| 内容方向 | 建议纳入报告的表达方式 | 当前是否具备基础 | 仍需补充内容 |
| --- | --- | --- | --- |
| 病害风险逻辑链条 | 不再只展示孢子数量，而是自动生成“监测到孢子异常上升 -> 判定相关病害风险升高 -> 提醒开展防治处置 -> 后续跟踪发病情况和控制效果”的说明段落 | 部分具备 | 需补病害类型映射关系、预警阈值、处置记录、发病率或复核结果 |
| 适应性管理记录 | 在月报、阶段报告中增加“预警触发与处置成效”专栏，将虫情或孢子预警与管护动作关联展示 | 部分具备 | 需建立人工录入或后台维护字段，记录处置时间、处置措施、责任单位、处置后结果 |
| 监测体系科学性说明 | 在报告开头固定增加方法说明，表述监测点布设逻辑、监测对象、指标完整性、监测频率和在线化能力 | 已具备 | 需整理成统一的模板文字，在各类报告中固定引用 |

### 8.1 病害风险逻辑链条建议写法

建议在孢子监测相关报告中，统一采用“数据发现 + 风险判断 + 管护动作 + 结果反馈”的规范表达。示例如下：

> 某时段内，系统监测到孢子捕捉数量明显升高，结合预设病害识别规则，判定纹枯病或稻瘟病发生风险上升。系统随即触发预警提示，建议开展针对性巡查与防治处置。后续结合现场核查和复测结果，对病害发生情况及防控效果进行跟踪评价，形成从监测预警到管护响应再到结果复核的完整闭环。

如需进一步形成正式考核表述，建议补充以下要素：

| 需补充项 | 用途 |
| --- | --- |
| 孢子类型与病害类型对应关系 | 用于把“监测数据”转为“病害风险判断” |
| 预警阈值或分级规则 | 用于形成自动判断逻辑 |
| 实际防治措施记录 | 用于证明系统已指导现场管护工作 |
| 发病率、巡查结论或复测结论 | 用于形成“结果反馈”闭环 |

### 8.2 适应性管理记录建议写法

建议在月报或阶段报告中，单独设置“适应性管理成效”专栏，统一按如下口径描述：

> X月X日，系统监测到某监测点虫情或病害风险指标异常升高，达到预警阈值。平台随即发出预警提示，管护单位据此开展巡查、精准施药、生物防治或现场清理等措施。后续监测结果显示，相关风险指标明显回落，表明监测预警已对现场管护形成有效支撑，体现出较好的适应性管理成效。

建议后台至少补充以下记录字段：

| 字段 | 用途 |
| --- | --- |
| 预警时间 | 对应触发时间点 |
| 预警对象 | 明确虫情、孢子或其他风险来源 |
| 预警级别 | 用于分级展示 |
| 处置措施 | 记录实际采取的管理动作 |
| 处置单位/责任人 | 体现责任落实 |
| 复核结果 | 证明措施执行后的效果 |

### 8.3 监测体系科学性说明建议写法

建议在所有自动化报告的开头或“方法说明”部分，固定加入类似表述：

> 本项目围绕水土流失、农业面源污染、病虫害等关键生态胁迫，构建了涵盖地表径流、雨量、水质、虫情、孢子等要素的在线监测体系。其中，地表径流监测布设了次生林、橡胶林、果林等不同生态类型监测点，具备对照分析基础；监测指标覆盖流速、流量、含沙量、雨量及主要水质因子，并实现连续在线采集，为项目成效评估、风险预警和适应性管理提供了持续的数据支撑。

## 九、特别说明

1. 当前农田排水水质系统实际保留的指标为：氨氮、总磷、总氮、高锰酸盐。
2. 当前业务口径已明确：现有“高锰酸盐”按 COD 替代表达。
3. 当前平台如需支撑“适应性管理闭环”正式考核，还应尽快补充预警日志、处置记录和结果复核三类业务数据。
4. 本表为“当前平台能力对照表”，主要用于项目汇报、需求梳理和实施边界确认，不直接替代正式验收报告。
