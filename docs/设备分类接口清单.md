# 设备分类接口清单

本文按以下 5 个设备类别整理接口：

1. 农田排水水质及面源污染负荷监测系统配套设备
2. 智能虫情测报灯
3. 孢子捕捉仪
4. 4G雨量计
5. 地表径流监测系统

每个类别都包含两部分：

- 本系统接口 URL：前端或第三方系统直接访问当前 FastAPI 服务使用
- 上游真实取数 URL：后端采集程序实际请求的设备平台接口

## 统一说明

### 本系统接口基础地址

```text
http://<host>:8001
```

### 上游平台 1：智慧农业平台

- 基础地址：`https://zhnlkj.com/iotSmasrt`
- 登录接口：`POST https://zhnlkj.com/iotSmasrt/login`
- 配置项 `PLATFORM_BASE_URL` 必须填写基础地址 `https://zhnlkj.com/iotSmasrt`，不要写成 `.../login`
- 后续采集接口需要在请求头中带 `Authorization: <token>`

登录请求体示例：

```json
{
  "username": "你的平台账号",
  "password": "你的平台密码"
}
```

### 上游平台 2：WHXPH 平台

- 基础地址：`https://iot.whxph.com:44300/XPHapiv2`
- 当前项目直接取最新数据接口：`GET https://iot.whxph.com:44300/XPHapiv2/data-n/{deviceId}`
- 该接口在当前代码里未附加鉴权参数

## 1. 农田排水水质及面源污染负荷监测系统配套设备

当前实际设备编码：

```text
16133028
```

### 本系统接口 URL

| 方法 | URL | 用途 | 参数 | 返回核心字段 |
| --- | --- | --- | --- | --- |
| GET | `/api/sensor/water_quality/daily` | 获取水质时间序列 | `days`，默认 `30`，范围 `7-90` | `date`, `cod`, `tn`, `tp`, `nh4n` |
| GET | `/api/summary/overview` | 获取大屏概览中的水质块 | 无 | `data.water_quality` |
| GET | `/api/analysis/guideline-metrics` | 获取《指南》适配后的水质削减率、径流估算、病虫闭环和气象支撑结果 | 无 | `data.water_quality`, `data.runoff_erosion`, `data.pest_management`, `data.weather_support` |

### 上游真实取数 URL

| 方法 | 真实 URL | 用途 |
| --- | --- | --- |
| GET | `https://iot.whxph.com:44300/XPHapiv2/data-n/16133028` | 获取农田排水水质设备最新一条原始数据 |

### 上游返回后落库字段

当前项目已经按现网真实设备口径瘦身为 4 个有效指标：

- `ammonia_nitrogen`
- `total_phosphorus`
- `total_nitrogen`
- `permanganate_index`

## 2. 智能虫情测报灯

当前实际设备编码：

```text
202603172301
```

### 本系统接口 URL

| 方法 | URL | 用途 | 参数 | 返回核心字段 |
| --- | --- | --- | --- | --- |
| GET | `/api/insect/latest` | 获取最新一条虫情数据 | 无 | `collection_time`, `total_count`, `species_data`, `image_url` |
| GET | `/api/insect/trend` | 获取虫情日趋势 | `days`，默认 `7`，范围 `1-30` | `date`, `total`, `species` |
| GET | `/api/insect/species-stats` | 获取虫种统计 | `days`，默认 `7`，范围 `1-30` | `[{name, value}]` |
| GET | `/api/insect/species-heatmap` | 获取虫种热力图数据 | `days`，默认 `14`，范围 `7-30` | `dates`, `species`, `values` |
| GET | `/api/insect/combined-trend` | 获取虫情与孢子联合趋势 | `days`，默认 `30`，范围 `7-90` | `date`, `insect`, `spore` |
| GET | `/api/summary/overview` | 获取综合概览中的虫情块 | 无 | `data.insect`, `data.insect_trend` |
| GET | `/api/summary/device-status` | 获取虫情设备在线状态 | 无 | `name`, `code`, `status`, `last_data` |

### 上游真实取数 URL

虫情和孢子共用同一个上游接口，只是 `code` 不同。

| 方法 | 真实 URL | 用途 |
| --- | --- | --- |
| GET | `https://zhnlkj.com/iotSmasrt/http/monitor/getBugWarmByCode?code=202603172301&collectionTime=` | 获取虫情数据，按需在 `collectionTime=` 后追加时间范围 |

说明：

- `{start}`、`{end}` 格式为 `YYYY-MM-DD HH:MM:SS`
- 当前代码采集窗口为最近 2 小时
- 请求头需要携带 `Authorization: <token>`

示例：

```text
https://zhnlkj.com/iotSmasrt/http/monitor/getBugWarmByCode?code=202603172301
```

如需按时间范围查询，在 `collectionTime=` 后追加：

```text
{start},{end}
```

## 8. 备注

- 虫情测报灯和孢子捕捉仪共用上游接口 `/http/monitor/getBugWarmByCode`
- 气象和土壤类设备使用的是 `/http/monitor/getSensorByCode`，但不在本次你指定的 5 类设备范围内
- 4G雨量计和地表径流监测系统目前都复用 WHXPH 的 `data-n/{deviceId}` 最新数据接口，只是后端提取字段不同
- 当前项目里没有单独的 `/api/sensor/water_quality/latest`，只有 `/api/sensor/water_quality/daily`
